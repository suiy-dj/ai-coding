## 三、融资与市场数据

### 融资情况（2024-2025）
- **Cognition Labs（Devin）**：2024 年 B 轮融资数亿美元，估值超过 20 亿美元
- **Adept AI**：融资超过 4 亿美元，投资方包括 Greylock、General Catalyst
- **Scale AI**：2024 年 E 轮融资，估值超过 140 亿美元（Agent 基础设施）

*注：具体融资数据以官方披露为准，部分数据为媒体报道估算*

### 市场规模预测
- Gartner：2026 年 AI Agent 市场规模约 50 亿美元，预计 2028 年达 250 亿
- 但分析师指出，目前 90% 的"Agent"只是包装过的 Chatbot

### 谁在赚钱
- **卖铲子的**：云厂商（AWS/Azure/GCP）、GPU 厂商（NVIDIA）
- **基础设施**：向量数据库（Pinecone）、Agent 框架（LangChain）
- **真正有收入的 Agent 公司**：不到 5%

### 谁在烧钱
- **大部分 Agent 初创公司**：收入几乎为零，靠融资续命
- **大厂 Agent 部门**：投入巨大，商业化路径模糊
- **原因**：技术不成熟 → 用户不愿意付费 → 无法形成正向循环

## 四、技术局限性（专家观点）

### 主要问题
1. **规划能力弱**：Agent 很难正确拆解多步骤任务，容易卡在中间步骤
2. **记忆问题**：金鱼记忆，记不住之前的操作和上下文
3. **容错率低**：遇到意外情况（弹窗、网络错误、界面变化）就崩溃
4. **幻觉问题**：会"以为"自己完成了任务，实际上根本没做
5. **安全风险**：自主操作可能误删文件、泄露数据、执行恶意操作

### 专家评价
- **吴恩达**："Agent 很有潜力，但需要克服可靠性和安全性问题才能大规模应用"
- **Yann LeCun**："目前的 LLM-based Agent 缺少世界模型，无法真正理解因果关系"
- **某大厂 AI 负责人（匿名）**："行业内都知道是泡沫，但谁都不敢先说出来"
- **Hacker News 高赞评论**："我们正处在'AI 之冬'的前夜，太多公司拿着 PPT 当产品卖"

## 五、正面案例（真正能用的 Agent）

### 案例 1：客服工单处理 Agent
- **能做什么**：自动分类用户工单、提取关键信息、生成初步回复
- **为什么能成**：场景边界清晰、任务标准化、容错空间大

### 案例 2：代码 Review Agent
- **能做什么**：检查代码规范、发现常见 bug、提出改进建议
- **为什么能成**：有明确的规则、不需要自主决策、人类最终把关

### 案例 3：数据分析 Agent（部分场景）
- **能做什么**：自动生成 SQL、执行数据查询、输出基础报表
- **为什么能成**：任务定义清晰、有验证机制、错误成本低
