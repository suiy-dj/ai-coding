# 自动化写作协作系统

基于 Claude Code 的自动化写作协作 Agent，让 AI 写出有"人味"的文章。

## 目录结构

```
/
├── CLAUDE.md                    # 核心协作指南
├── README.md                    # 本文件
├── briefs/                      # 需求简报
│   └── template.md              # 简报模板
├── knowledge_base/              # 知识库
│   └── template.md              # 知识库模板
├── style_guide/                 # 风格指南
│   ├── 我的风格.md               # 个人风格说明
│   └── 去 AI 味检查清单.md         # 审校清单
├── material/                    # 个人素材库
│   └── README.md                # 素材说明
├── projects/                    # 进行中的项目
├── images/                      # 图片资源
└── published/                   # 已发布文章
```

```
开始新文章
```

### 修改已有文章

```
修改这篇文章
```

### 去 AI 味

```
去 AI 味
```

### 查资料
```
查一下 [主题] 的信息
```

## 写作流程（9 步）

1. **理解需求 & 撰写 Brief** - 明确写作目标
2. **信息收集** - 收集最新资料
3. **选题与大纲** - 提供 3-4 个选题方案
4. **创建协作文档** - 复杂任务需要
5. **学习用户风格** - 读取风格指南
6. **使用个人素材库** - 去 AI 味核心
7. **开始写作** - 用用户视角写作
8. **多重审校** - 事实 + 去 AI 味 + 细节
9. **配图处理** - 如需要

## 核心原则

1. **Think Aloud** - AI 每步说明思考过程
2. **强制事实核查** - 涉及新概念必须核查
3. **选题优先** - 先确认选题再写正文
4. **个人素材库** - 使用真实经历降 AI 味
5. **多重审校** - 系统删减至少 30% 废话

## 禁止事项

以下情况 AI 不应自行决定：
- 跳过事实核查步骤
- 使用过时信息
- 跳过 Think Aloud
- 跳过强制审校
- 未经用户确认直接写正文

## 配置

编辑 `.claude/settings.local.json` 添加需要的权限。

## 使用方法

1. 首次使用，填写 `style_guide/我的风格.md`
2. 在 `material/` 目录中添加你的真实经历和观点
3. 每次写作前，在 `briefs/` 创建简报
4. 写作过程中随时调用去 AI 味检查清单
